### 目前存在问题

1. Windows 下人脸和声音模型都能跑通，也就是四种载荷在都能够正确释放。测试的流程是 Windows 下打开网易云音乐 (会创建一个子进程执行恶意程序 ---> 捕获人脸或声音 ---> 正确识别释放载荷)
2. Android 目前人脸是通过的，四种载荷都可以释放。声音部分没有测试，但是释放逻辑是类似的，代码逻辑应该没有什么问题 (但是还得多测试一些其他用例)

存在问题：

1. 人脸识别模型和声音识别模型的精确率都比较低 (60 左右) | 准确率和攻击成功率比较高 (如过没有记错的话)。
2. 声音识别模型静默状态下会有概率直接释放载荷，也就是误触 (上一次存在的问题)。
3. Android 下的模型本身就是量化过的，也就是将 Windows 下的模型参数转化为 Android 下可以加载的 tensorflow-lite 文件，参数会有精度损失，这也可能是误报率比较高的原因之一。
4. 人脸识别模型训练过程比较慢
5. 对于 Android 来说，训练集中的目标应该有多个角度的照片，但是在软件上捕获的人脸图片是单角度的 (因为 Windows 攻击机是固定的)，可能到时候需要用手机拍摄多个角度的照片再上传到攻击机上，然后由软件再上传到训练服务器上。
